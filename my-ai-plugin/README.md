# AI 助理 - VS Code 插件

基于 OpenAI 兼容 API 的 AI 助理，支持 DeepSeek、豆包、GPT 等多模型，提供代码解释、Bug 修复、代码优化、续写和单测生成等功能，主交互入口固定在编辑器右侧聊天面板。

## 功能一览

### 核心功能
- **多模型支持**：配置多个 AI 模型，快速切换（DeepSeek / 豆包 / OpenAI / Claude 等）
- **三种工作模式**：Code（可修改文件）、Ask（只读对话）、Plan（先规划后执行）
- **流式对话**：实时显示 AI 回复，支持 SSE 流式传输
- **工具调用**：AI 可读写文件、列出目录，自动执行多轮工具调用
- **折叠过程展示**：生成中与完成后都可查看结构化执行过程摘要

### 代码操作（右键菜单 + 快捷键）
| 功能 | 快捷键 |
|------|--------|
| 解释代码 | `Alt+E` |
| 修复代码 | `Alt+F` |
| 优化代码 | 右键菜单 |
| 续写代码 | 右键菜单 |
| 生成单测 | 右键菜单 |

### 多会话管理
- **右侧聊天主入口**：聊天面板固定在编辑器右侧区域，`Alt+Q` 可打开并聚焦输入框
- **会话 Tab 栏**：在当前聊天面板顶部标签切换多个独立对话，单击切换、双击重命名、`×` 删除
- **新建对话 / 新建标签**：`Alt+N` 在当前聊天标签打开会话启动器；`Alt+T` 在当前聊天面板新增聊天标签，首次调用会先打开聊天面板
- **持久化**：所有会话重启后自动恢复，旧数据无缝迁移

### 聊天增强
- **@ Mentions**：输入 `@` 或点击 `+` 菜单，统一下拉搜索工作区文件，同名文件显示相对路径
- **图片上传**：支持点击选择、拖拽到输入框、`Ctrl+V` 粘贴截图三种方式；缩略图预览；自动识别视觉模型
- **工作流触发**：扫描 `.windsurf/workflows/` 目录，QuickPick 选择并二次确认副作用后执行
- **终端错误分析**：在终端复制错误文本，点击工具栏按钮一键发给 AI 分析
- **重新生成**：每条 AI 回复底部显示「↺ 重新生成」按钮，删除旧回复后重新调用
- **拖拽文件**：从资源管理器拖拽文件到输入区域，自动添加为上下文
- **Slash 命令**：输入 `/` 触发快捷命令（`/explain` `/fix` `/optimize` `/test` `/complete` `/clear`）
- **聊天内搜索**：`Ctrl+F` 搜索对话内容，高亮匹配，上下导航
- **对话历史持久化**：重启 VS Code 后对话不丢失
- **输入历史回溯**：`↑` / `↓` 箭头浏览之前的输入
- **导出对话**：一键导出为 Markdown 文件
- **上下文占用估算**：实时显示已使用 / 上限 / 百分比，并按阈值进行颜色提醒
- **输入框字符计数**：实时显示输入字符数，超长时变色警告

### UI 体验
- **代码块增强**：语法高亮 + 复制/插入按钮 + 行号显示（>5 行时）
- **Markdown 表格渲染**：支持对齐方式

### 快捷键
| 快捷键 | 功能 |
|--------|------|
| `Alt+Q` | 打开并聚焦右侧聊天输入框 |
| `Alt+M` | 切换工作模式（Code → Ask → Plan） |
| `Alt+N` | 在当前聊天标签打开新对话启动器 |
| `Alt+T` | 在当前聊天面板新增聊天标签 |
| `Ctrl+F` | 搜索对话内容 |
| `Escape` | 停止 AI 生成 / 关闭搜索 |
| `↑` / `↓` | 浏览输入历史 |

### 智能特性
- **状态栏模型指示器**：底部显示当前模型名，切换时自动同步
- **项目上下文感知**：自动读取工作区 `README.md`（前 2000 字）和 `package.json`（关键字段）注入系统提示词
- **项目类型自动检测**：识别 Node.js/TypeScript/Python/Go/Rust 等技术栈，增强 AI 上下文
- **Git 状态感知**：自动检测当前分支和未提交变更文件，注入 AI 上下文
- **自定义系统 Prompt**：`myAiPlugin.systemPrompt` 配置项，非空时完全覆盖内置提示词
- **视觉模型支持**：自动识别 GPT-4o、Claude 3、Gemini 等视觉模型，启用图片上传
- **上下文窗口管理**：长对话自动截断旧消息，防止 Token 超限
- **网络代理支持**：HTTP/HTTPS 代理（CONNECT 隧道），兼容 VS Code `http.proxy` 配置
- **API 双重超时**：连接超时（30s）+ 流式空闲超时（60s）
- **启动配置校验**：缺少 API Key 时自动提示
- **Webview 状态持久化**：切换标签页后输入内容和滚动位置不丢失

## 配置

在 VS Code 设置中搜索 `myAiPlugin`：

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `myAiPlugin.models` | AI 模型列表（数组） | DeepSeek Chat |
| `myAiPlugin.activeModelIndex` | 当前激活的模型索引 | 0 |
| `myAiPlugin.maxTokens` | 最大输出 Token 数 | 4096 |
| `myAiPlugin.temperature` | 温度参数 | 0.3 |
| `myAiPlugin.language` | 回复语言 | 中文 |
| `myAiPlugin.proxy` | 代理地址（如 `http://127.0.0.1:7890`） | 空 |
| `myAiPlugin.systemPrompt` | 自定义系统提示词（非空时覆盖内置提示词） | 空 |
| `myAiPlugin.panelTitle` | 聊天面板标题（可自定义为企业名称） | AI 助理 |

模型配置项支持可选的 `contextWindow` 字段，用于估算当前会话的上下文占用和颜色阈值提醒。

### 模型配置示例

```json
{
  "myAiPlugin.models": [
    {
      "name": "DeepSeek Chat",
      "modelId": "deepseek-chat",
      "baseUrl": "https://api.deepseek.com",
      "apiKey": "sk-xxx",
      "contextWindow": 64000
    },
    {
      "name": "GPT-4o",
      "modelId": "gpt-4o",
      "baseUrl": "https://api.openai.com",
      "apiKey": "sk-xxx",
      "contextWindow": 128000
    }
  ]
}
```
